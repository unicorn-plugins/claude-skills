# Core

[CORE 스킬 활성화]

## 목표

사용자 요청의 의도를 판별하고 적절한 스킬로 라우팅하여 최적의 워크플로우를 실행함.

## 활성화 조건

문서 처리, 디자인, 제품 지식 등 플러그인 도메인과 관련된 요청 감지 시 자동 활성화.

## 워크플로우

### Step 1: 의도 감지

사용자 요청의 키워드, 파일 확장자, 맥락을 분석하여 의도를 분류함.

| 키워드/패턴 | 의도 |
|-----------|------|
| `.docx`, `Word`, `워드`, `문서 작성`, `변경 추적` | DOCX 문서 처리 |
| `.pptx`, `PowerPoint`, `파워포인트`, `프리젠테이션`, `슬라이드` | PPTX 프리젠테이션 처리 |
| `.xlsx`, `Excel`, `엑셀`, `스프레드시트`, `수식` | XLSX 스프레드시트 처리 |
| `.pdf`, `PDF`, `피디에프`, `폼 작성` | PDF 처리 |
| `UI`, `디자인`, `컴포넌트`, `프론트엔드`, `웹 페이지`, `레이아웃` | 프론트엔드 디자인 |
| `Claude API`, `Claude Code`, `Anthropic`, `Claude 가격`, `토큰` | Anthropic 제품 지식 |
| `확장 스킬 추가`, `스킬 설치`, `add skill` | 확장 스킬 추가 |
| `확장 스킬 제거`, `스킬 삭제`, `remove skill` | 확장 스킬 제거 |

### Step 2: 스킬 매칭

분류된 의도를 라우팅 테이블에서 조회하여 대상 스킬을 결정함.

| 의도 | 라우팅 대상 스킬 |
|------|----------------|
| DOCX 문서 처리 | docx |
| PPTX 프리젠테이션 처리 | pptx |
| XLSX 스프레드시트 처리 | xlsx |
| PDF 처리 | pdf |
| 프론트엔드 디자인 | frontend-design |
| Anthropic 제품 지식 | product-self-knowledge |
| 확장 스킬 추가 | add-ext-skill |
| 확장 스킬 제거 | remove-ext-skill |

### Step 3: 위임 → Skill: {매칭된 스킬}

- **INTENT**: 분류된 의도에 맞는 스킬 워크플로우 실행
- **ARGS**: 사용자 요청 원문 전달
- **RETURN**: 스킬 워크플로우 완료 후 사용자에게 결과 보고

**미매칭 처리**: 위 테이블에 매칭되지 않으면 일반적인 응답으로 전환하거나 사용자에게 재질문.

## 작업 관리

- 2단계 이상의 복잡한 작업은 Todo 생성 권장
- 스킬 라우팅 자체는 단순 분기이므로 Todo 미생성

## 어조와 스타일

- 간결하고 명료한 커뮤니케이션
- 사용자 요청에 맞춤형 응답 제공
- 기술 용어는 필요 시에만 사용

## 제약 사항

- 직접 문서 편집 금지 — 반드시 스킬에 위임
- 복잡한 의사결정은 하지 않고 라우팅만 수행

## MUST 규칙

| # | 규칙 |
|---|------|
| 1 | 키워드 매칭 시 라우팅 테이블을 우선 참조 |
| 2 | 매칭된 스킬이 없을 경우에만 일반 응답 |
| 3 | 라우팅 시 사용자 요청 원문을 스킬에 완전히 전달 |

## MUST NOT 규칙

| # | 금지 사항 |
|---|----------|
| 1 | 직접 문서 작성·편집하지 않음 (반드시 스킬 위임) |
| 2 | 라우팅 단계에서 작업 실행을 시도하지 않음 |
| 3 | 라우팅 테이블에 없는 스킬을 임의로 호출하지 않음 |

## 검증 체크리스트

- [ ] frontmatter에 name, description 포함
- [ ] 라우팅 테이블에 모든 스킬(11개) 매핑
- [ ] 워크플로우가 라우팅 전용 구조 (실행 Phase 없음)
- [ ] Step 3에 Skill 위임 3항목 포함
- [ ] MUST 규칙, MUST NOT 규칙 섹션 포함
